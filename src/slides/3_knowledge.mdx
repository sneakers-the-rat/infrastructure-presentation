# Thinking expansively

* A common lack in all these tools is that they don't really have a clear means for
people to communicate about their use, to interact with the shared components.
* Relatedly, our embrace of heterogeneity and federation has also left us with
a bunch of disordered schemas, what to do?

Here's three steps on a gradient for software
* Traditional software library -- code is fixed, but you can create local files. like MS paint
* Extensible software library -- certain features can be added in
* Modular software library -- the basic components of the system can be swapped out
* ??? -- natural final step is some way of discussing and governing the module combinations

And in schemas
* Traditional centralized schama -- there is one vocabulary
* Extensible schema -- there is a core vocabulary and a way of making new terms
* Modular schema -- core links can be replaced by links with comparable semantic structure
* ??? -- a way of negotiating the links between data and meaning of terms


My argument: investment in collaborative design and communication tools is the missing link
